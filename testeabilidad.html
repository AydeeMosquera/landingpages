TESTEABILIDAD

DEFINICION:

Es la facilidad que tiene un sistema de realizar pruebas en un contesto dado. ya que si el nivel de la prueba es alta, seria mas facil encontrar las fallas en el sistema mediante pruebas.
normalmente algunos se pueden probar y otros no.
estas pruebas se pueden vincular con la dificultad con la que sean encontradas las fallas potenciales en un sistema, para evaluar esta capacidad de prueba se necesita formar un conjunto y ya que estas pruebas se realicen seran aplicadas en cada caso y se puede determinar con algunas especificaciones ya que si el sistema es pqueño la capacidad de prueba sera alta.
Modelo de pueba en el que un programa procesa una entrada y produce una salida.



ESCENARIO DE PROBABILIDAD GENERAL

Entre estos tenemos:

FUENTE DE ESTIMULO:
Estas pruebas se realizan por medio de probadores de unidades, de integridad o de sistemas.

ESTIMULO:
Se ejecutan con un conjunto de pruebas ya que debido a la finalizacion del incremento de codificacion ya sea una capa de clase o de servicio este para la integracion completa de un subsistema.

ARTEFACTO:
Ya sea una unidad un subsistema o un sistema que se este probando.

RESPUESTA:
El sistema puede ser controlado y de esta manera realizar las pruebas que se desean hacer, para observar los resultados de la prueba que se realizo.

TACTICAS PARA LA TESTABILIDAD:

El objetivo de esta tactica es la capacidad de realizar pruebas permitiendo de esta manera pruebas mas faciles.

PROBABILIDADES: Se aplica en componentes de esfuerzo y eficacia de las pruebas de software.
PROPIEDADES: Es el tamaño, la complejidad y la capacidad con la que realizan de las pruebas.
METODOS: Son aquellos que se utilizan en las pruebas. Los modulos se determinan por sus factores.
CONTROL: Es el control que se de en el momento en que lo requieran las pruebas.
OBSERVAR: Observar cada resultado de las pruebas ya sean intermedios y finales.
CAPACIDAD DE AISLAMIENTO: Es el grado en el que sea sometidas estas pruebas.



CONTROLAR Y OBSERVAR LAS PRUEBAS

Estos elementos son fundamentales para la capacidad de prueba ya que al realizar las pruebas en el sistema se pueden solucionar muchas fallas que se encuentren en el programa.
Estos dos elementos estan muy unidos y son muy parecidos ya que no tiene sentido controlar algo si no se puede observar lo que esta sucediendo en el sistema.
Si se controla facilmente los datos de entrada se facilita la prueba.
• Caja de Area: Es aquella que nos permite aislar una instancia en el sistema del mundo real.
Un sistema complejo es muy dificil de probar ya que debido al gran espacio de estado en el que tiene lugar los calculos y un numero mayor de interconexiones entre elementos del sistema.
Debemos recordar que hay pruebas que se pueden solucionar por que son faciles de encontrar a simple vista y otras que son encontradas cuando el software es instalado. Entre mas grande sea el software estas pruebas no se podran hacer manualmente.
