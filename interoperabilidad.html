INTEROPERABILIDAD

DEFINICION:

Es el grado en el que dos o mas sistemas pueden intercambiar informacion util a traves de interfaces en un contexto paticular.
Como el poder interpretar de manera correcta los datos que se intercambian de forma aislada.
Estas se pueden ver afectadas por un sistema que se interoperen. Ya que un sistema que trabaje junto no puede cometer errores.

RAZONES PARA QUE UN SISTEMA INTEROPEREN

* El sistema proporciona un servicio para ser utilizado por una coleccion de sistemas desconocidos.

* Construir capacidades a partir de sistemas existentes.



ESCENARIOS DE INTEROPERABILIDAD:

* Un sistema inicia una solicitud de interoperabilidad con otro sistema.
* Solicitud para intercambiar informacion con otros sistemas.
* Los sistemas que interoperan se ven cuando el tiempo de ejecucion se presente antes o despues del tiempo de ejecucion.
* Cuando la informacion se entiende de forma receptora de manera sintactica y semantica.
* En cuanto al porcentaje de informacion sea correctamente procesada o de manera incorrecta.
* EJEMPLO: A traves de un soap, un api rest se puede presentar un escenario cambio de informacion ya que tanto el frontend y el backend se realizan por separado.



TACTICAS PARA LA INTEROPERABILIDAD

Se identifican en dos categorias que son.
* Localizar.
* Administrar interfaces.
* Las categorias: es descubrir el sistema. Cuando los sistemas interactuan en tiempo de ejecucion este se puede localizar mediante la busqueda de un sistema de un directorio desconocido.
* Agregar o eliminar capacidades de una interfaz.
• OBJETIVO: Es realizar intercambio de informacion intencionalmente o rechazar solicitudes de intercambio de informacion.
* Los sistemas se ubican entre si y luego administren las interfaces para que estas puedan realizar el intercambio de informacion.
