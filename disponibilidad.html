DISPONIBILIDAD

DEFINICION:

Propiedades de software que esta alli y lista para llevar su tarea a cabo cuando la necesita.
Este sistema puede tener la capacidad de enmarcar o reparar fallas, la disponibilidad trata de minimizar el tiempo de interrupcion del servicio mediante la mitigacion de fallas, las fallas pueden ser internas o externas y se pueden realizar de forma automatica, dichas fallas se pueden reparar de manera manual o automaticas.

ESCENARIOS DE DISPONIBILIDAD

* Diferentes origenes tanto internos como externos.

* Se pueden producir fallos: de omision el cual no responde a una entrada.

* Crash: sufre fallos de emision repetidas.

Tiempo: cuando la respuesta se da rapido o tarda mucho.



Alta disponibilidad
ANALISIS DEL ARBOL DE FALLOS:

Especifica un estado del sistema el cual afectan de forma negativa, la confiabilidad o la seguridad de nuestro programa.



TACTICAS DE DISPONIBILIDAD.

Estas tacticas se encuentran diseñadas cuando el presenta fallas.
Como: fallas de detencion, de recuperacion y de prevencion.
Esta tactica pretende tolerar los errores de especificacion mediante el uso de especificaciones de requerimientos separeados.
• OBJETIVO: El sistema debe estar disponible para la coordinacion de las fallas que se puedan presentar.
