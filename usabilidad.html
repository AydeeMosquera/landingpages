USABILIDAD

DEFINICION:

La usabilidad esta relaciona con lo fácil que es para el usuario poder realizar una tarea deseada y el tipo de asistencia al usuario el cual proporciona el sistema. Ya que a lo largo de los años, este enfoque tambien le da una facilidad de uso que se ha revelado como una de las formas más económicas y sencillas de mejorar la calidad de un sistema (o, precisamente, la percepción de calidad del usuario).
Ya que el usuario puede interactuar de una manera facil y eficiente con el sistema dandole tareas especificas.

AREAS QUE COMPREDE LA USABILIDAD

Alguna de esas son:

FUNCIONES DEL SISTEMA DE APRENDIZAJE:
El usuario no esta familiarizado con un sistema en particular o con un aspecto en particular.

USO EFICIENTE DE UN SISTEMA:
Se puede incluir la capacidad con la que el usuario se redirije al sistema despues de haber emitido un comando.

MINIMIZAR EL IMPACTO:
Los usuarios podrian cancelar un comando emitido icorrectamente o minimizar vel impacto de los errores.

ADAPTAR EL SISTEMA:
El sistema puede ser adaptado a las necesidades del usuario para que este se adapte de una manera facil a una tarea que el usuario le de.

ESCENARIO GENERAL DE USABILIDAD:

Estas son algunas las proporciones de los escenarios generales de usabilidad:

FUENTE DE ESTIMULO:
Ya que el usuario final pueda tener un rol especifico, tales como administrador del sistema o de la red.

MEDIO AMBIENTE:
Son las acciones que le conciernen al usuario la facilidad de un uso ocurrido en el tiempo de ejecucion.

ARTEFACTO:
Es la parte especifica del sistema con el cual el usuario pueda interartuar.

RESPUESTA:
El sistema le proporciona al usuario funciones necesarias o de anticipacion a las necesidades del usuario.



OBJETIVO:

En esta tactica el objetivo es cancelar un comando de ejecucion prolongada o deshacer un comando completado donde el usuario pueda realizar una tarea con facilidad y agregar datos y comandos.
EJEMPLO:
El usuario descarga una nueva aplicacion y la utiliza de manera productiva.
